# ID-backed native Pi changes

## Goal

Replace Grove's branch-naming and ZMX runtime machinery with a smaller Pi-native model: every Grove Change has one immutable Grove-owned ID, an ID-named Git branch, one self-contained capsule under `~/.grove`, native Pi JSONL sessions, and one stable human title inferred asynchronously from the first Pi session. Preserve the valuable real-Git creation-base and destructive-removal safety while substantially shrinking the command surface, implementation, bundled assets, and compiled-CLI test scaffolding.

## Approach

Make a clean break around an ID-backed Change capsule. Reuse the existing 32-lowercase-hex Grove identity generation pattern, but promote it to an atomically reserved `ChangeId` that is independent of Pi session IDs and is also the exact local Git branch name. Store the worktree, Pi-native session directory, Grove record, and final analytics artifacts below one immutable capsule path. Grove launches Pi directly and blocks until it exits; Pi itself supplies conversation persistence through `--session-dir` and `--continue`. A small extension links native Pi JSONLs to the Change and starts an isolated `pi --print` title generator without delaying the real agent turn.

Keep the existing integration detection and compare-and-delete safety, but move Grove-owned lineage facts out of branch-keyed Git config and into the capsule record so all Grove state survives branch deletion in `~/.grove`. Archive the final worktree state before destructive removal using a temporary Git index, rather than trying to reconstruct code changes from Pi tool calls.

This is deliberately a breaking pre-1.0 migration: do not auto-import old pending worktrees, ZMX sessions, branch-keyed lineage config, or `~/.local/state/grove` data, and never delete that old data automatically. Reconsider that choice if Grove has user data that cannot reasonably be recreated or manually retained.

## Key Decisions

- A **Change** is Grove's durable task. Its immutable `ChangeId` is a 32-character lowercase hexadecimal value generated by Grove, atomically reserved as a capsule directory, and used verbatim as its local Git branch. The ID is independent of every Pi session ID and is normally hidden from users.
- A **Title** is the only human-facing name. The first successfully inferred native Pi session title initializes the Change title; later native Pi sessions get their own persisted Pi names but never retitle the Change or rename Git.
- There is no semantic branch name, prompt-derived branch slug, branch rename, worktree move, pending detached state, or user rename command.
- `grove new` and `grove switch` accept no positional selector. `switch` always uses the existing terminal picker, populated only by active Grove capsules. `remove` likewise accepts no positional branch; it targets the current managed Change or opens the picker from the primary checkout. `--from`, `--shell`, `--force`, `list`, and `init` remain.
- Plain managed `new`/`switch` launch Pi and emit no shell-navigation directive, so the calling shell remains where it started after Pi exits. `--shell` is the only path that writes a directory directive. `grove init fish|zsh` therefore remains required for parent-shell navigation.
- A manually started `pi` is unmanaged. Grove does not globally install its extension, wrap the user's `pi`, or promise to capture arbitrary Pi sessions; managed Pi is always launched explicitly by Grove with its extension and capsule session directory.
- Direct Pi replaces ZMX; there is no detach, attach, background persistence, active-session query, or process termination. A per-capsule advisory lock held for the direct Pi child prevents two writers and makes launch/removal refuse while managed Pi is open, including forced removal.
- Pi JSONL is the canonical conversation/usage/tool history. Grove never copies, renames, exports, or normalizes it. The extension appends one versioned Grove custom entry linking each native session to its `ChangeId`.
- Automatic naming is an explicit part of managed Pi usage and may make an additional provider request. The generator is fire-and-forget, isolated with `--print --no-session --no-tools --no-context-files --no-skills --no-extensions`, validates a single three-to-four-word line, and cannot block or fail the real agent turn.
- The naming call sends the first substantial prompt to Pi's configured naming model a second time. This privacy/cost behavior must be documented in help and README rather than hidden behind the extension.
- The title generator runs once for every unnamed native Pi session. Generator failure leaves the session/Change unnamed; the picker uses `Untitled` plus a short ID only when necessary for disambiguation. There is no continuous sentiment tracking or automatic retitling.
- Grove captures an authoritative base-to-final Git patch/stat snapshot before branch/worktree deletion. Pi tool history is analytics input, not a code-state authority.

## Contracts

### CLI

```text
grove new [--from REF] [--shell]
grove switch [--shell]
grove list
grove remove [--force]
grove init fish|zsh
```

- `new --shell` creates an untitled Change and navigates without requiring or launching Pi.
- Managed `new` validates Pi before mutation, creates the Change, then launches Pi directly.
- `switch` and `switch --shell` always require a TTY picker; unmanaged primary, linked, and detached Git worktrees are not candidates.
- `remove` uses the current managed Change when invoked inside one; from the primary checkout it uses the same capsule picker. It never accepts an opaque ID as a normal human selector.
- Picker rows use the stable Change title, or `Untitled`; duplicate/untitled rows may show a short ID secondarily but selection always returns the full internal ID.
- After managed Pi exits, Grove does not change the calling shell's directory. Shell navigation occurs only after successful `--shell` or removal of the current worktree.

### Capsule

```text
~/.grove/<repo-name>-<common-dir-digest>/<change-id>/
  change.json
  worktree/
  sessions/
    pi/
      <Pi-native timestamp>_<Pi-native session-id>.jsonl
  artifacts/
    change.patch
    stats.json
```

`change.json` is versioned and contains only Grove-owned facts: full Change ID, optional stable title, active/closing/archived state, creation time, repository locator, creation base ref/OID/parent, and optional closure time/outcome/tip/target/integration classification. Branch is derived as `ChangeId`; Pi session metadata remains native in `sessions/pi`.

Capsule directories are private (`0700`) and Grove-owned records/artifacts are created with private file modes (`0600`) on Unix because both Pi transcripts and patches can contain source, prompts, tool output, and secrets.

The capsule path never moves. Archival removes only `worktree/` and the Git branch after artifacts are durable, leaving Pi's absolute `cwd` and `parentSession` paths as stable as possible. Recreating `worktree/` at that same location is the only supported future resume path for an archived session.

### Managed Pi

```text
cwd: <capsule>/worktree
argv: pi --session-dir <capsule>/sessions/pi --continue --extension <materialized Grove extension>
```

The extension handles `startup`, `new`, and `fork` session lifecycles, appends a `grove` custom entry if absent, and arms first-interactive-prompt naming only when the current native session has no persisted Pi name. It must compare the captured native session ID before applying an asynchronous title so a late result cannot name a replacement session.

The extension starts a hidden `grove __title` subprocess without awaiting it, passing the captured Change/native-session identity and first prompt. That subprocess owns the isolated `pi --print` invocation and atomic Change-record update, then returns one validated title on stdout. The extension calls Pi's native `setSessionName()` only if the captured native session is still current. Malformed output, generator failure, or Pi exit is non-fatal; metadata writes remain inside Grove rather than being reimplemented in TypeScript.

### Removal artifacts

Before destructive removal, Grove creates a temporary alternate Git index, seeds it from the recorded creation base, stages the entire current worktree with `git add -A`, writes a snapshot tree, and generates a binary-capable `change.patch` plus `stats.json` from the recorded base to that tree. This includes committed, staged, unstaged, deleted, renamed, binary, and untracked non-ignored content without modifying the real index.

Artifacts are installed durably before Git mutation. Grove then revalidates the expected branch OID, removes the worktree, compare-and-deletes the ID branch, and marks the record archived with `integrated` or `discarded`. Any validation/artifact failure leaves the branch and worktree untouched; a failure after artifact installation leaves the record recoverably `closing` rather than falsely archived.

## File Map

- `src/change.rs` — new deep capsule module: `ChangeId`, versioned record, repository/capsule paths, atomic allocation/load/enumeration/title/closure updates, and picker-facing Change views.
- `src/git.rs` — retain deep real-Git workflows, creation-base resolution, integration detection, final snapshot capture, and compare-and-delete; replace pending/semantic-branch/session-ID plumbing with explicit ID-branch worktrees at capsule paths and record-backed lineage.
- `src/session.rs` — shrink to the Pi adapter: executable validation, advisory lock, extension materialization under `~/.grove/runtime`, direct blocking launch/resume, and title-generator invocation. Delete every ZMX/runtime/attach/list/kill path.
- `src/pi-extension.ts` — link native sessions to Change IDs, react to native session lifecycle, start per-session fire-and-forget naming, and persist names through Pi's native session interface; never rename Git or move worktrees.
- `src/main.rs` — remove branch positionals/value completion/hidden branch-renaming command; orchestrate capsule-backed `new`, picker-only `switch`/`remove`, title-based list rendering, `--shell`-only navigation, and the private `__title` subprocess entrypoint.
- `src/shell.fish`, `src/shell.zsh` — retain the parent-shell directive wrapper and command/flag completion; remove assumptions about positional branch completion.
- `Cargo.toml`, `Cargo.lock` — remove `flate2` and `tar`; retain `blake3`, Serde, Clap completion, and Crossterm.
- `vendor/zmx/` — delete all bundled archives, license, and runtime documentation.
- `tests/cli.rs` — replace ZMX/pending/branch-argument tests with a small set of coherent capsule, picker, native-Pi, title, lineage, archive, and destructive-safety workflows.
- `tests/support/mod.rs`, `tests/support/agent.sh` — remove runtime socket/PID/detach/kill/pending helpers; add capsule discovery and a fake Pi that distinguishes managed interactive launch from isolated `--print` and writes minimal native-shaped JSONL.
- `README.md`, `VISION.md`, `AGENTS.md` — document the new Change/Title/PiSession language, picker-only UX, `--shell`/`init` behavior, extra naming request, native session ownership, capsule layout, direct-process limitations, and archive safety.
- `CONTEXT.md`, `docs/adr/0001-id-backed-native-pi-changes.md` — record the resolved glossary and the hard-to-reverse identity/storage/runtime decision.
- `docs/research/herdr-evaluation.md` — mark the earlier multiplexer/ID-UX recommendation as superseded by the no-process-persistence decision; keep it as historical research rather than an active design contract.

## Out of Scope

- Migrating or deleting existing Grove worktrees, semantic branches, branch-keyed lineage config, ZMX processes, or `~/.local/state/grove` sessions.
- Terminal/process persistence, detach/reattach, laptop-sleep execution, PTY hosting, or building any multiplexer.
- Globally installing/wrapping Pi or capturing manually launched Pi sessions.
- Supporting Claude, Codex, other agents, multiple simultaneous writers, Cloudflare sandboxes, remote handoff, or online execution; the independent Change/session identity and capsule layout only keep those options open.
- Continuous title sentiment tracking, automatic Change retitling, manual rename, semantic branch aliases, title arguments/completion, or non-picker Change selection.
- Configurable naming providers/models/prompts, retry queues, or blocking on title generation. The first version uses Pi's configured default in one isolated best-effort call.
- Analytics queries, dashboards, databases, uploads, retention policy, or full forensic command-output capture. This change only preserves native Pi data and authoritative Git artifacts in an analytics-friendly layout.
- Commit/push/PR publication semantics for opaque ID branches.
